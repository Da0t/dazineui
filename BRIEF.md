dazineui — Motion Primitives for AI-Built UI
Project Brief

---

## The Problem

AI coding assistants produce functional code — but consistently fail at visual quality. Ask any AI to "add a hero with animation" and you get `motion.div` fade-ins, `hover:scale-105`, a flat blue gradient, maybe a broken Three.js sphere. It's not that the AI can't write Three.js — it's that it has no curated reference. It improvises, and improvised animation is always slop.

---

## The Solution

dazineui is an npm package that ships two things:

1. **Importable primitives** — six senior-level Three.js / WebGL components that cover the most common hero/background design patterns
2. **AI rules files** — dropped automatically on install, teach Cursor and Claude Code when to use each primitive, how to route vague prompts, and what to refuse to generate

```bash
npm install dazineui
```

After install, the AI in your project already knows the primitives. It doesn't need to be told. It doesn't need to improvise. It picks the right component, imports it, and configures it correctly.

---

## Distribution

Two ways to install:

**Per-project (npm):**
```bash
npm install dazineui
```
Drops `.cursor/rules/dazineui-motion.mdc` via postinstall. Works immediately in any Cursor or Claude Code session in that project.

**Global (all projects):**
```bash
npx dazineui setup
```
Installs to `~/.claude/skills/dazineui/` and `~/.cursor/skills/dazineui/`. The AI uses it in every project on your machine without a per-project install.

---

## How the AI Integration Works

Cursor reads every `.mdc` file in `.cursor/rules/` and attaches it to every AI conversation automatically. The installed `dazineui-motion.mdc` contains:

- **Routing table**: maps "make it pop" → `FlowGradient preset="stripe"`, "neon glow" → `SpotlightGradient`, etc.
- **Decision flow**: the AI checks MANIFEST before writing any visual code
- **Hard rules**: never use blur as the primary glow technique, always tune materials, always add post-processing
- **Anti-pattern list**: the exact patterns to refuse — `transition: all`, `hover:scale-105`, CSS keyframes as 3D stand-ins

The AI doesn't ask for instructions. It knows.

## The Primitives

Six ship in v0.1.0. All dark-first, WebGL/Canvas-based, no blur:

| Primitive | Visual character |
|---|---|
| `FlowGradient` | 5 ribbon meshes, sine-displaced, additive blending. Stripe-style diagonal waves. |
| `SpotlightGradient` | Neon TubeGeometry along CatmullRom curves with Bloom. Linear/cyberpunk. |
| `MeshGradient` | 120 thin bezier fiber curves, Canvas 2D. Woven texture, organic depth. |
| `NoiseGradient` | 2000 flow-field particles, sin+cos vector field. Generative art. |
| `AuroraGradient` | Two dense particle sheets crossing in 3D. Dot matrix aurora. |
| `WaveGrid` | 50×50 perspective particle grid, mouse-interactive. Classic Three.js waves. |

Roadmap includes: MorphOrb, CrystalShard, HolographicGrid, ParticleText, ChromaText, GalaxyField.

## Technical Stack

- **React** (≥18), peer dependency
- **Three.js** (≥0.160), peer dependency
- **@react-three/fiber** + **@react-three/drei** + **@react-three/postprocessing**, peer dependencies
- **tsup** for building — outputs CJS + ESM + type declarations
- **TypeScript** with strict mode, full prop types on every primitive
- **`'use client'`** on all components — Next.js App Router safe when wrapped in `dynamic(..., { ssr: false })`
- **No bundled runtime dependencies** — consumers provide React/Three

```
scripts/
  postinstall.js        ← drops .cursor/rules/dazineui-motion.mdc on npm install
.cursor/rules/
  motion-rules.mdc      ← the rules file that ships with the package
app/                    ← Next.js demo app (not published to npm)
dist/                   ← built output (generated by tsup, published to npm)
```

## Design Philosophy

1. **No blur as a primary visual ingredient.** Glow comes from HDR-emissive materials + Bloom postprocessing.
2. **Real geometry carries meaning.** PlaneGeometry ribbons, TubeGeometry tubes, Points particles — not CSS pseudo-elements dressed as 3D.
3. **Additive blending on dark backgrounds.** This is how real light mixes — not `opacity: 0.5`.
4. **Speed props are multipliers, never absolute time values.** Consumers set `speed={0.5}` not `duration={2000}`.
5. **Post-processing as a finishing pass, not a crutch.**
6. **Senior-level means: the AI learned from real Three.js designers.** Not from Stack Overflow snippets or tutorial sphere examples.
